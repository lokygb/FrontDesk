User Feedback

Testing Interface
  • Difficult to ascertain exact status of grading job
  • Unable to correct submissions that were handed in improperly and hence cannot be auto-tested by handin.
  • Confusing parameters to create job page. Common questions include, What is the reference? Do I need to select a comparator?
  • Performance on larger tests
Testing Setup
  • Limited flexibility in testing, do more than just string comparison
  • Sometimes returns 0 for matching output
  • Ease of use for testing setup
Grading/Execution Interface
  • Extracting Results
    • Unable to view the entire section in a summarized view.
    • In order to print results of entire section, one needs to print each student in the section one at a time. Would like to just hit print and get all student results on separate pages.
    • Spreadsheet export for Excel grading managements.
  • Unable to edit student code in any way. It would be nice if one could make marks in the student code for style/correctness issues and have them show up in the printout
  • Ability to modify objective score, and the addition of a subjective and extra points block for each assignment.
Student Interface
  • Unclear if submission went through. Most of the time compile is disabled and pretests don't exist. Would be nice if email was sent or some sort of secondary confirmation existed.
  • More detail from a compile error. Listing the errors is not useful if you think the program compiles. It would be nice to give a snapshot of the directory struture, current path, and then the compiler erros in order to reveal to the student possible directory structure.
  • Manage user submission directory. In case of small error, it would nice to fix on handin side. File management is ad hoc.
  • Performance lags during busy handin times (deadline).
